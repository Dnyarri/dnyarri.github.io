ScaleNx

Scale2x, Scale3x, Scale2xSFX and Scale3xSFX image scaling in pure Python

ScaleNx is a group of Pixel-art scaling algorithms, intended to rescale images without blurring sharp edges, which makes it totally different from regular interpolation like bilinear and bicubic you see in Photoshop or GIMP.

Scale2x and Scale3x algorithms (sometimes referred to as AdvMAME2x and AdvMAME3x) were initially developed by Andrea Mazzoleni for scaling up small pixel graphics like game sprites and icons without introducing new colors and blurring contrast edges. That's what it was invented for, and nothing else.

Later on, Scale2xSFX and Scale3xSFX were introduced for the same goal, providing different diagonals rendering and eliminating artifacts initial ScaleNx produce on some patterns.

Currently very efficient ScaleNx implementations are numerous, but it seems like all of them are developed for screen viewing only, like DOS emulators for old low-color games, etc. It seems that nobody gives a value to rescaling an arbitrary image in arbitrary color mode, and saving the result as a file in some common widespread format. Everybody seem to use ScaleNx only for its initial quite limited task.

However, it appears that ScaleNx may be successfully misused for something absolutely different, like scaling up low resolution text scans before OCR, or drawings before tracing them to vectors, or simply improving some scans of very old printed materials. ScaleNx capability of keeping things sharp and paying attention on shapes may give very interesting results for a wide variety of images.

With that in mind I created general purpose (i.e. working with any image of any color mode) pure Python implementation of ScaleNx algorithms. By saying «pure Python» I mean it - it doesn't even use bytearrays, only lists of lists of lists. Consider it an exacting spiritual practice or whatever. Anyway, pure Python nature allows current ScaleNx implementation to be run under Windows, Linux, MacOS, or practically anything you can find a Python for.

You can download ScaleNx as fully functional application for single and batch images rescaling, as well as pure ScaleNx module to include it into your own software, or just to laugh at it.

ScaleNx package includes suitable human-ready GUI shell, not surprisingly called ScaleNxGUI.py, providing easy launch of single PNG, PPM or PGM (PBM support is limited to reading only, result may be saved to another format like PNG) file rescaling procedure, or batch rescaling of all images (in forementioned formats) in chosen folder and subfolders. Note that batch processing uses asynchronous multiprocessing, greatly reducing run time for a price of devouring next to 100% of CPU power, taking RAM for several images simultaneously, and rendering GUI rather unresponsive, looking like it hung up. Don't worry, it can't hang up, it can only possibly crash when trying to process really huge images (although for most recent version, implementing some memory-saving tricks, I yet have to see it).

Needless to say, PNG support in ScaleNxGUI is provided with PyPNG, and PPM and PGM support with PyPNM; both are pure Python modules, thus making the whole program quite cross-platform. There is even Python 3.4-compatible build (see below), tested and proven to work under Windows XP 32-bit on a twenty years old notebook. Practically, the only problem with Windows XP 32-bit in the year 2025 appeared to be finding a web browser correctly displaying GitHub pages; Supermium appeared to be capable enough.

Update: In version 25.11.15.1 visual shell program VisualNxGUI.py added to main branch. Program provides a preview for single file rescaling, quick saving with Ctrl+S allows sequential rescaling. Be careful though: during such a sequential rescaling image size grows geometrically, and soon enough will exceed your computer memory. Although later on, in VisualNxGUI ver. 26.2.11.7, memory usage for auxiliary processes (i.e., image display) was reduced to minimum, for a huge images even a minimum is quite substantial. For upscaling really big images, it is strongly recommended to use ScaleNxGUI main shell - it doesn't display anything but buttons and menus, and therefore eat your precious memory for processing only.

All modules updated to most current version. PyPI release for developers updated to ScaleNx 2025.11.15.34

Previously on ScaleNx: - In main ScaleNx version 25.09.25.09 and it's Python 3.4 companion version 25.09.25.34 further GUI legibility improvements are introduced.

PNG compression and PPM/PGM type (binary or ASCII) settings may be configured via GUI option menus and saved by Ctrl+Clicking on info bar as JSON "scalenx.ini" file in your user directory (directory address get copied to clipboard so you may paste it into your file manager and instantly get right to this directory, wherever it is).

Preferences file may be edited with any text editor, and loaded back to ScaleNx with Alt+Click to info bar (for toads and lesser species in dementia all these hints appear on mouse over info bar).

Speaking of dementia, version 25.08.22.34 was intentionally downgraded from pathlib to os, making it even more compatible with old Python version. Now you can Use the Source™ and see how ladies were laid and recursive directory reading was made before Path.rglob was introduced.

Both main and .py34 versions are relinked with newest PyPNM version 2, providing considerable memory usage reduction while working with PPM and PGM files.

Version 25.02.09.18 of main ScaleNxGUI program have no change in ScaleNx module but is relinked with updated version of PyPNM module providing per row file writing, thus reducing memory usage; PyPNG data exchange changed to row generators thus making memory usage as small as possible with this format.

On my rather old notebook main ScaleNxGUI program appeared to be able to upscale 8224x8224 px RGB image with Scale2x and Scale2xSFX up to 16448x16448 px, and then successfully save the result as both PNG and PPM. Considering the fact that resulting image will occupy ca. 1.4x1.4 meters when printed at 300 ppi, I assume that further scaling up the experiments with larger images on my notebook may raise doubts in my formally confirmed «mentally sane» status.

In version 25.02.01 FIR optimization of Scale2x, Scale3x, Scale2xSFX and Scale3xSFX performed, speed gain is in range from ca. 15% for Scale2x to ca. 50% for the others (since Scale2x work on minimal kernel already, there's not much room for further list read access reduction, therefore speed increase is minimal).

For Windows users, compiled ScaleNxGUI32.exe for Windows XP 32 bit and ScaleNxGUI64.exe for Windows 10 64 bit are available for download at newest release page, what as to other platforms, since the program is pure Python and quite self-contained, release source is supposed to work with any current canonic Python installation right out of the box.

As a side note, it's worth mentioning that, when working with PNG, ScaleNxGUI.py changes PNG resolution (if present) so that image print size remains unchanged. You may consider using this fact for batch increasing resolution for documents with linked images (or, with some manual intervention, for container documents).
